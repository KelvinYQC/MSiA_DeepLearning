Model Archetctures¶

LSTM layer: The LSTM layer is added to the model using the LSTM function. It has 32 units, which determines the dimensionality of the output space. The input_shape parameter is passed to the first LSTM layer to define the shape of the input data. LSTM(64): This is the second LSTM layer in the model. It has 64 units. Since return_sequences is not specified, it defaults to False, meaning that this layer will not return the output sequence.

Dense layer: After the LSTM layer,Dense(128) is a fully connected dense layer with 128 units. It takes the output of the previous layer (which is the output of the last LSTM layer) and applies a linear transformation to produce a new representation. Then a Dense layer with 1 unit is added. This layer is responsible for generating the final output of the model.

Compilation: The model is compiled using the compile method. The loss function is set to 'mean_squared_error', which is suitable for regression tasks. The optimizer is set to 'adam', which is a popular optimization algorithm.

Training: The model is trained using the fit method, where X_train and y_train are used as the training data. The model is trained for 50 epochs with a batch size of 16.

Loss function (mean_squared_error): The loss function used in the model is mean squared error (MSE). MSE is a common choice for regression tasks, where the goal is to minimize the difference between predicted and true values. It squares the differences between predicted and true values, giving higher weights to larger errors. The choice of MSE is suitable for this regression problem of predicting the sine wave values.

Optimizer (adam): The optimizer used in the model is Adam. Adam is an adaptive learning rate optimization algorithm that combines ideas from both AdaGrad and RMSProp. It dynamically adjusts the learning rate during training, allowing for faster convergence and better handling of different learning rates for different parameters. Adam is a popular choice for deep learning models as it often yields good results with relatively less manual tuning.

Hypreparameters¶

LSTM_3 units (32): The number of LSTM units determines the dimensionality of the output space of the LSTM layer. In this case, 32 LSTM units have been chosen. The choice of the number of units depends on the complexity of the problem. Increasing the number of units allows the model to capture more complex patterns in the data but also increases the computational requirements and potential overfitting risk. The choice of 32 units strikes a balance between complexity and efficiency for this specific problem.

Number of epochs (50): The number of epochs determines how many times the model will iterate over the entire training dataset during training. Each epoch consists of forward and backward passes through the network, updating the weights. The choice of the number of epochs depends on the complexity of the problem, the size of the dataset, and the convergence of the model. In this case, 50 epochs have been chosen, which allows the model to make multiple passes over the data while not being too computationally expensive.

Batch size (16): The batch size specifies the number of samples used in each forward and backward pass during training. It determines the number of samples that will be propagated through the network before updating the weights. The choice of batch size affects the trade-off between computational efficiency and model convergence. Smaller batch sizes can provide more accurate weight updates but can be slower due to more frequent weight updates. In this case, a batch size of 16 has been chosen, which is a relatively small batch size suitable for this problem size.

Number of parameters¶

Number of parameters = Number of units in the layer * Number of inputs to each unit + Number of biases in the layer

lstm_3 (LSTM):

The LSTM layer has 32 units. Since return_sequences=True, the output shape will have a sequence length of 1. Hence, the output shape is (None, 1, 32). The number of parameters is calculated as follows: Input size: The input shape is (None, X_train.shape[1], X_train.shape[2]) = (None, Timesteps, Input_dim), where Timesteps is the length of the sequence and Input_dim is the number of features. For the LSTM layer, the number of parameters is calculated as: 4 [(Input_dim + 32) 32 + 32] = 4,352. The 4 represents the four gates in an LSTM unit (input gate, forget gate, output gate, and cell gate). lstm_4 (LSTM):

The LSTM layer has 64 units. Since return_sequences is not specified (defaults to False), the output shape will be a single vector of length 64. The number of parameters is calculated similarly to lstm_3: 4 [(32 + 64) 64 + 64] = 24,832. The input shape is the output shape of lstm_3.

dense_5 (Dense): The dense layer has 128 units. The input shape is the output shape of lstm_4, which is (None, 64). The number of parameters is calculated as: (64 * 128) + 128 = 8,320. The first term represents the weights connecting the input to the dense layer, and the second term represents the biases.

dense_6 (Dense): The dense layer has 1 unit. The input shape is the output shape of dense_5, which is (None, 128). The number of parameters is calculated as: (128 * 1) + 1 = 129. Similar to dense_5, the first term represents the weights, and the second term represents the bias. The dense layer has 1 unit, with 32 inputs (the hidden state from the LSTM layer), plus 1 bias, for a total of 33 parameters.
